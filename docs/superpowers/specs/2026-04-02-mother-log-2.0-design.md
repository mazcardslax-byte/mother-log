## Design System — Carbon Amber

### Color tokens (CSS custom properties in `index.css`)

| Token | Value | Use |
|-------|-------|-----|
| `--bg` | `#0a0a0a` | Page background |
| `--surface` | `#111111` | Card / sheet surface |
| `--border` | `#2a2418` | Card borders, dividers |
| `--amber` | `#fbbf24` | Primary accent, active tab, stat numbers |
| `--amber-2` | `#fcd34d` | Secondary amber |
| `--amber-3` | `#fde68a` | Tertiary amber |
| `--amber-bg` | `#2a1f00` | Amber tinted backgrounds (active tab pill, badges) |
| `--amber-border` | `#3a2e00` | Amber tinted borders |
| `--text-primary` | `#f5f5f0` | Primary text |
| `--text-body` | `#c5b08a` | Body / secondary text |
| `--text-muted` | `#6a5a3a` | Muted labels, timestamps |

### Typography

- Font: DM Sans (existing) — no change
- Page label: 11px, 700 weight, uppercase, 0.12em letter-spacing, `--amber`
- Page title: 20px, 700 weight, `--text-primary`
- Card title: 13–15px, 600 weight, `--text-primary`
- Stat number: 18–22px, 700 weight, `--amber`
- Section label: 9px, 700 weight, uppercase, 0.1em letter-spacing, `--text-muted`
- Timestamps / meta: 10px, `--text-muted`

### Card system

Two-level surface:
- Page bg: `--bg` (`#0a0a0a`)
- Card surface: `--surface` (`#111111`) + `1px solid --border`
- Card border-radius: `10px` (large cards), `8px` (stat chips)

### Event history left-border accents (kept, semantic)

| Event type | Left border color |
|------------|-------------------|
| clone | `#fbbf24` (amber) |
| amendment | `#7c3aed` (violet) |
| feeding | `#16a34a` (green) |
| transplant | `#0284c7` (sky) |
| reduction | `#dc2626` (red) |

### Tab bar

- Container: `--surface` background, `1px solid --border`, `12px` radius
- Active pill: `--amber-bg` background, `--amber` text, `9px` inner radius
- Inactive: `--text-muted` text
- Active state communicated by color + background pill (no icon-only)

### Page header pattern (consistent across all tabs)

```
[amber label — uppercase]     [optional badge/action]
[page title — large]
```

### Stat strip pattern (consistent across tabs)

3-column grid of stat chips: large amber number + uppercase muted label below.

## Bins Tab (new)

**Purpose:** Inventory of sealed totes containing fully dried flower.

**Data model** — stored in existing `dryroom` Supabase key under `bins: []`:

```js
{
  id: string,        // uuid
  strain: string,    // from STRAINS list
  dateSealed: string, // ISO date
  toteLabel: string, // e.g. "Tote #1"
  notes: string      // optional
}
```

**No new Supabase key.** Reads/writes to `dryroom.bins`. Uses same `subscribeToKey` / `saveToDB` pattern as DryRoomTab.

**UI:**
- Page header: "Inventory" label + "Bins" title + "+ Add Bin" button top-right
- Summary strip: 2 stat chips — tote count + strain count
- List: one card per bin showing strain name, tote label, date sealed, chevron
- Empty state: dashed border card with "+ Add another bin"
- Add/edit: modal or inline form — strain (dropdown from STRAINS), tote label (text), date sealed (date picker), notes (optional text)
- Delete: long-press card → confirmation dialog (swipe-to-delete avoided, unreliable in PWA)

**Tab bar:** "Bins" added as 8th tab with a `Package` icon (Lucide).
